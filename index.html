Observer
The observer pattern is a behavioral pattern which defines a subscription mechanism to notify objects about events that happen to the object they are observing. I followed a tutorial in David Baron's Hands-On Game Development Patterns with Unity 2019. I used this pattern to add events to Cubethon, such as the player changing color at certain distances.

The visitor pattern is a behavioural design pattern which seperates algorithms from objects on which they operate. I followed the tutorial in David Baron's Game Development Patterns with Unity 2021. Spatial Partition
The spatial partition pattern is a pattern which splits up a given area into a grid to more effeciently identify the nearest objects. I followed a tutorial provided by Erik Nordeus. In addition to this, I convereted the project into a three dimensional space and added an fps counter to track the difference between when the pattern is active or inactive.

Game Systems 1


Heroborn
Heroborn is a first person shooter in which your goal is to collect the four coins in the castle. This game uses an audio mixer with on screen sliders to adjust volume midgame and postprocessing effects including grain and vignette. In addition to that, it uses a trail renderer on the bullet and counters in C# to delay how often you can shoot. Heroborn uses render textures and a secondary camera to create a minimap.

Cubethon
Cubethon is a runner style game in which the player avoids obsticles as it travels forward. This game uses instantiation to create more obsticles in random locations as the game continues, allowing for an infinte level. It also makes use of post processing effects such as depth of field, bloom, motion blur, and lens distortion.

Terrible Tweeters
Terrible Tweeters is a 2d game very similar to Angry Birds. The game uses an audio mixer and on screen sliders to control volume. It also uses a line renderer to trace the path the bird will follow upon release and basic animation through out the game.
